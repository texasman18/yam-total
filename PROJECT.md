# YAM 통합 앱 — 프로젝트 종합 문서

> **얌(YAM) — 내 주변 화장실.** 한국·일본·대만 3개국을 하나로 통합한 화장실 찾기 웹앱.
> "한국에서는 내 주변 화장실 앱, 일본·대만에 도착하면 조작 없이 그 나라의 한국어 화장실 지도로 바뀌는 앱."

- 최종 정리일: 2026-07-19
- 소스 저장소(개발/테스트): https://github.com/texasman18/yam-total → yam-total.vercel.app
- 본 서비스(웹·안드로이드·iOS): https://github.com/texasman18/yam-toilet → **yam-toilet.vercel.app**
- 로컬 정본 폴더: `~/Documents/Claude/통합 yam/`

## 1. 제품 정의

| 모드 | 대상 | UI | 지도 | 데이터 | 진입 |
|---|---|---|---|---|---|
| 🇰🇷 한국(기본) | 한국인 + 방한 외국인 | 한국어 ↔ 영어 | 카카오(ko) / 구글(en) | 행안부 **52,480건** | 앱 실행 기본 |
| 🇯🇵 일본 | 방일 한국인 | 한국어 고정 | 구글(한국어 라벨) | OSM+지자체 **8,835건** | 카드 탭 / GPS 자동 |
| 🇹🇼 대만 | 방대만 한국인 | 한국어 고정 | 구글(한국어 라벨) | 환경부+OSM **17,561건** | 카드 탭 / GPS 자동 |

**합계 78,876곳.** 해외 데이터는 해당 모드 진입 시에만 lazy-load(한국 사용자 성능 영향 0).

### 절대 원칙
1. 기본 화면은 항상 한국. 한국 사용자 경험 불변.
2. 해외 진입은 두 갈래(국기 카드 탭 / GPS 자동), 결과는 동일.
3. 자동보다 사용자의 손이 우선(수동 선택 세션엔 GPS 자동전환 OFF — `sessionStorage yam_manual`).

## 2. 기술 아키텍처 (단일 파일 `index.html`, 약 1,640줄)

- **상태 2축**: `curLang`(ko|en, 한국 모드에서만 토글) × `curCountry`(kr|jp|tw)
- **단일 진실 공급원** `COUNTRIES` 테이블(국가별 dataFile·region·placesQueries·bbox·defaultView·토스트)
- **모든 전환은 `setCountry(cc,{manual,silent,gpsLat,gpsLng})` 하나로 수렴**
- **GPS 자동 전환**: `countryOf(lat,lng)` bbox 판정(한국 최우선) → `maybeAutoSwitch` → 현지 좌표로 지도 점프
- **구글 지도 동적 로더**: JP/TW·한국Places = `language=ko`, 한국 영어 = `language=en`. 언어 불일치 시에만 1회 리로드. 한국↔일본/대만 전환은 리로드 없음.
- **한국 검색 = 카카오 + 행안부 공공데이터 + 구글 Places**(항상 포함, `locationBias`로 세션 이력 무관 일관성)
- **실시간 위치 추적**: `watchPosition`으로 이동 시 내 위치 마커 갱신
- **외부 리소스 0**(폰트·CDN 미사용, 시스템 폰트) — 지도 SDK 2종만 예외. 다크모드·reduced-motion 대응.
- **디자인**: Airbnb 스타일(브랜드색 #ff385c). 랜딩=기존 구조 유지+하단 일본/대만 국기 카드. 지도 화면 상단바 한 줄(검색창 말줄임), 해외 모드 한/영 토글 회색 비활성화.

### 파일 구조 (배포 대상)
```
index.html          단일 앱 코드
sw.js               서비스워커(yam-total-v3, 해외 번들 캐시 제외)
manifest.json       PWA(테마 레드, icon.png)
icon.png            앱 아이콘
toilets.json        한국 데이터        japan_toilets.json / taiwan_toilets.json  해외 번들
*_sources.md        출처 고지          vercel.json  rewrites+캐시 헤더
pipeline/{japan,taiwan}/  데이터 재생성 파이프라인(배포 무관)
docs/               운영 문서          마케팅/  ASO·마케팅 자료
```
※ 본 서비스(yam-toilet)에는 위에 더해 **앱 필수 파일 보존**: `.well-known/assetlinks.json`(안드로이드 TWA)·`privacy.html`(스토어 심사)·`netlify.toml`·기존 `yam-japan/`.

## 3. 배포 & 플랫폼 전략

- **웹·안드로이드·iOS 동일**: yam-toilet은 `server.url` 방식이라 **웹만 갱신하면 세 플랫폼 자동 반영**(재심사 불필요).
- 배포 흐름: 로컬 `통합 yam` 수정 → yam-total push(테스트) → 검증 후 YAM 저장소 동기화 → yam-toilet push(본 서비스).
- **지도 키**: yam-toilet 도메인은 카카오·구글 키에 등록돼 있어 지도 완전 작동. (테스트 도메인 추가 시 두 콘솔에 도메인 등록 필요 — `docs/지도키등록_및_yam-toilet이관.md`)
- 스토어 앱 등록(안드로이드·iOS)은 **약 1개월 후** 예정. 그때까지 주간 점검(`docs/테스트_모니터링_계획.md`).

## 4. 검증 완료 항목 (라이브 실측)

- KR: 강남 40개(카카오12+구글10+공공18), 내 위치 핀 복귀, 검색·상세·길찾기
- JP: 신주쿠·도톤보리 자동 전환, 건물 단위 한국어 라벨 지도
- TW: 타이베이역·타오위안공항 자동 전환, 위생등급 뱃지, 원문 병기
- **VPN 없이 GPS 좌표 주입**으로 오사카 도톤보리→일본·타오위안공항→대만 자동 전환 라이브 확인
- 시나리오 A~F(권한거부·수동우선·귀국복귀·딥링크·JP↔TW 무리로드) 전수 통과, 콘솔 에러 0

## 6. 개발 이력 (주요 커밋)

초기 구축 → 랜딩 복원+일본/대만 지도 카드 → 전체 화면·가운데 정렬 → 내 위치 버튼 버그 수정 → 한국 검색 일관성(구글 Places+공공데이터 항상 포함) → Airbnb 디자인 적용 → 위치 아이콘 변경 → 전체 검수(색 통일·위치필 버그) → **본 서비스 yam-toilet 이관** → 일본·대만 줌 15→17(한국과 라벨 밀도 일치) → 실시간 GPS 추적 통합.

## 7. 알려진 주의사항 (다음 세션 인수인계)

- **yam-toilet에 다른 세션/사용자의 직접 커밋 가능** → index.html 통째 복사 전 반드시 `git log`로 원격 신규 커밋 확인·병합(과거 watchPosition 덮어쓴 사고).
- **로컬 테스트 시 서비스워커가 이전 index.html 캐시** → 검증 전 SW unregister + caches.delete.
- **TW 데이터 재생성은 `pipeline/taiwan/.venv` 파이썬**으로만(hanja 폴백 누락 방지).
- 카카오 키는 localhost:3000만 등록 → 로컬 지도 테스트는 3000 포트.
