# 3개국 화장실 데이터 연 1회 갱신 절차 (매년 12월 25일)

- 원칙: **매년 12월 25일** 3개국 번들을 최신으로 재생성해 커밋한다. (사용자 결정, 2026-07-18)
- 갱신 후 반드시 §4 검증을 통과시킨 뒤 push한다. 웹 push만으로 iOS 앱에 자동 반영된다(server.url 방식).
- 갱신 알림: 매년 12월 20일경 Claude 세션에서 "/YAM 데이터 갱신" 요청 또는 캘린더 반복 일정 등록 권장.

## 1. 한국 `toilets.json` (행정안전부 공공데이터)
- 소스: 공공데이터포털(data.go.kr) "전국공중화장실표준데이터"
- 절차: 본 앱(YAM 저장소)의 기존 생성 방식과 동일 — 표준데이터 CSV 다운로드 → 기존 스키마 `{n,a,la,ln,t,h,p}`로 변환. (본 앱에서 갱신되면 그 파일을 복사해와도 된다 — 정본은 YAM 저장소)

## 2. 일본 `japan_toilets.json` (OSM)
```bash
cd "통합 yam/pipeline/japan"
rm -f raw_*.json          # 도시별 재수집하려면 raw 캐시 삭제
python3 fetch_osm.py      # Overpass API — 10개 도시 bbox
python3 merge.py          # 30m 격자 중복제거 + 검증 → ../../japan_toilets.json 경로 확인
```
- 주의: Overpass 무료 서버라 도시 사이 대기시간이 걸릴 수 있음(재시도 로직 내장). 출력 경로가 저장소 루트의 `japan_toilets.json`인지 실행 전 스크립트 상단에서 확인.

## 3. 대만 `taiwan_toilets.json` (환경부 + OSM)
```bash
cd "통합 yam/pipeline/taiwan"
python3 fetch_moenv.py    # 환경부 FAC_P_07 — curl 기반(SSL 검증 이슈 우회 내장)
python3 fetch_osm.py      # OSM 전국 보조
python3 merge.py          # 건물 그룹핑 + 30m 중복제거 + 등급(g) 부여
# ⚠️ translate.py는 반드시 hanja가 설치된 파이썬으로 실행할 것.
#    시스템 python으로 돌리면 hanja 폴백이 조용히 빠져 한자가 잔존한다 (실사고 이력).
python3 -m venv .venv && .venv/bin/pip install hanja   # 최초 1회
.venv/bin/python3 translate.py
```
- 환경부 API 키: 현재 data.gov.tw 배포 URL 내장 공개 키 사용 — 장기 운영 시 개인 키 교체 권장(`taiwan_sources.md` 참조).

## 4. 갱신 후 검증 체크리스트 (전 항목 통과 후 push)
- [ ] 3개 JSON 파싱 성공 + 건수 출력: `python3 -c "import json;[print(f,len(json.load(open(f)))) for f in ['toilets.json','japan_toilets.json','taiwan_toilets.json']]"`
- [ ] 건수가 전년 대비 ±30% 이내 (급감하면 소스 장애 의심 — push 금지)
- [ ] 대만 한글화율 점검: k·b 필드에 한자 잔존 없는지 표본 확인
- [ ] 좌표 범위: 각국 bbox 밖 좌표 0건
- [ ] 랜딩 신뢰 라인 건수 문구(`index.html`의 "78,000+")와 스토어 설명 수치 동기화
